又到年末总结时

时光匆匆，又到年末总结时。各 App 也开始了年度报告，可以预见下一周会被年度报告轰炸。

今年我以周刊为契机重新做起了公众号，真的非常感谢之前关注的朋友，即便有一段时间没更新也没放弃我。不仅如此，也有幸结识了很不错的朋友。

下半年买了无弦吉他后，弹唱之心便躁动了起来。开始在某B某音某书上传一些自娱自乐的弹唱视频。

当然也犯过错误（当时承诺的回顾也还没写）也亲历了公司裁员。由于想说的东西很多，就不在周刊里展开了。后面会专门写一些总结的文章回顾。

AI

1、justlovemaki/AIClient-2-API[^1]

标签：Tools

打破 AI Client 之间的隔阂，让 AI Client 中模型可以被其他应用使用。比如让 Cherry-Studio 可以用上Gemini 3.0 Pro、Qwen3 Coder Plus 等高级模型。

相当于是在两个 AI 客户端之间做了代理，最大限度地利用上各家模型。项目自带一个管理后台，也有另一个项目做了 UI 界面。

proxycast: https://github.com/aiclientproxy/proxycast

标签：Life

关注的博主 So!azy 推荐了一些个性化主页，作为个人展示的聚合入口。

其中这个是国内的作者所做，页面非常简单和干净。我也趁此机会做了一个自己的主页，可以扫码看看。

2、Speed Comparison - Programming Languages[^5]

标签：Coding

比较了 62 种编程语言的速度。C++ (clang++) 果然还是最快的，Rust (nightly) 排在第 7 位，Go 在 16 位仅次于 Rust（非 nightly） 。JavaScript（Bun）在 24 位，Node.js 在 44 位比 Java 略高 1 位。

标签：Coding,Tools

一个总结 Github 活动的网页工具。只要输入 Github 的用户名，就会拉取信息生成汇总。所以理论上你可以看其他人的活动情况。

下面是我 24 - 25 年的汇总。今年开始做了周刊，很明显能看出周日的活动数很多。

4、为什么我如此珍视善意的评论[^7]

标签：Life

作者在点评上收到了一个略带傲慢的评论后，由此展开的思考。通过分析负面评论的傲慢与无知，作者认为，善意的互动是信息交换，而非认知打压。对于主动发帖分享经验的作者，我们应心存感激和体谅。

在如今的网络环境下，哪怕你只是单纯地给别人留一句温暖的鼓励，都可能会遭遇其他人的恶语相向。所以，那些依然愿意在网络上向陌生人释放善意的人，是非常勇敢且值得欣赏的。

Coding

1、Introducing CSS Grid Lanes[^3]

标签：前端,CSS

介绍了 CSS 的 Grid Lens 属性，可以利用这一属性更加方便地实现漂亮的瀑布流式页面。浏览器会根据设置的 Grid 的 grid-template-columns 自动计算，然后填满页面内容。这个特性在 Safari 的 Preview 234 中实现。

2、Nature’s many attempts to evolve a Nostr[^4]

标签：架构

作者从服务器架构的一步一步推演，总结出了中继式的服务器可能是最终的演变结果。主要围绕着用户数据的所有权展开，只要数据的加密手段不在用户手上。那么用户就会像平民随时被挡在城外。

无论是最早的中心化服务（单独形成垄断），再到服务联邦（由多个寡头控制）都存在上述的问题。而 P2P（端到端）通信又存在技术成本和节点质量上的困难，并且也无可避免地会出现”中心“节点。既然避免不了”中心“节点的出现，那么就让数据加密掌握在用户手上，中心节点只做通信，从而形成中继式服务架构。

好一出分久必合，合久必分的演变过程。
